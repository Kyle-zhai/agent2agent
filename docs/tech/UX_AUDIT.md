# UX 审计（对标飞书）

> [!summary]
> 用户指令：「一切需要从用户使用角度出发」，以飞书的使用感为参照。本文是 2026-06-10 真机
> 审计（桌面 1280 / 平板 768 / 手机 375 三视口 + 关键旅程走查）的结论：**为什么飞书好用、
> 我们差在哪、这批修了什么、剩下什么**。状态以 [[FEATURES]] 为准。

## 一、飞书好用的本质（提炼为可执行原则）

1. **输入零摩擦**：Enter 即发送、Shift+Enter 换行；切走再回来草稿还在；中文输入法候选确认的
   Enter 永远不会误发。
2. **内容即所见**：文档/表格/图片点开就是排版好的阅读视图，不是源码；图片点击放大。
3. **永远有下一步**：每个面板有关闭、上一个/下一个、返回；空状态告诉你能做什么。
4. **小屏不残废**：手机上列表↔会话是栈式切换，浮层默认收起。
5. **反馈即时且可消除**：操作结果就地提示，自动消失，不打断。

## 二、本轮发现的问题（按用户旅程）

### 聊天（最高频旅程）
| 问题 | 严重度 | 状态 |
|---|---|---|
| Enter 不发送（仅 ⌘↵），与飞书习惯相反，新用户第一条消息就卡住 | 高 | ✅ 本批修复（含 IME 防误发） |
| 切换会话丢草稿 | 高 | ✅ 本批修复（localStorage per conv） |
| 图片无点击放大 | 中 | ✅ 本批修复（原生 dialog lightbox） |
| 错误提示靠 ?error= 横幅，不可关闭、跳转才消失 | 中 | ✅ 本批修复（可关 + 8s 自动淡出） |
| 无消息 Pin、无线程视图、无已读回执、表情仅 5 个 | 中 | backlog |

### 文件查看（Files）
| 问题 | 严重度 | 状态 |
|---|---|---|
| 查看器无 上一个/下一个，逐个看文件要来回点列表 | 中 | ✅ 本批修复（‹ Prev / Next › + n/N） |
| 打开子文件夹里的文件时，文件夹折叠着，看不到高亮行 | 中 | ✅ 本批修复（含所开文件的文件夹自动展开） |
| 无全页阅读模式（长文档在内嵌面板里读） | 低 | backlog |
| 文件行无"最后修改人/时间"（需按文件溯源 snapshot，成本较高） | 低 | backlog |

### 移动端（375px 实测截图：ux-chat-mobile）
| 问题 | 严重度 | 状态 |
|---|---|---|
| **聊天室整体被挤出屏幕**（rail + 固定 268px 列表占满宽度） | 致命 | ✅ 本批急救（响应式让位） |
| 私聊 dock 默认展开盖住大半屏 | 致命 | ✅ 本批急救（窄屏默认收起） |
| 完整的移动端导航（列表↔会话栈式、底部导航） | 高 | backlog（独立工程） |

### 其他
- 搜索无过滤器（按人/时间/类型）、会话内搜索无 UI —— backlog
- 通知粒度全有或全无（不能"仅 @ 我"）—— backlog
- Toast 体系（替代 query-param 错误横幅的全局方案）—— backlog

## 三、本批交付（v0.24 UX pass）

1. **Enter 发送 / Shift+Enter 换行**——主 composer、thinking 输入、私聊 dock 三处统一；
   **全部带 IME 守卫**（`isComposing` / keyCode 229：中文候选确认的 Enter 绝不发送）；⌘↵ 保留。
2. **草稿保存**——按会话存 localStorage，切换/刷新不丢，发送成功即清。
3. **图片 lightbox**——点击放大（≤90vw/85vh），文件名 + 下载，Esc/背景点击/✕ 关闭。
4. **查看器导航**——Prev/Next 循环（Finder 顺序）+ 位置 n/N + 含开文件的文件夹自动展开。
5. **移动端急救**——dock 窄屏默认最小化；shell 列表面板窄屏全宽、会话页让位给聊天室；
   桌面布局零变化（mobile-first 响应式类）。
6. **错误横幅**——✕ 可关 + 8 秒自动淡出（respect `prefers-reduced-motion`）。

## 四、Backlog（按 影响×成本 排序，未承诺）

1. **移动端完整改造**（列表↔会话栈式导航 + 底部 tab）——使用感的下一个台阶，独立工程
2. **Toast 反馈体系**——全局即时反馈，替代 query-param 横幅
3. **消息 Pin + 未读分隔线**（"以下为新消息"）
4. **搜索过滤器** + 会话内搜索 UI
5. **通知粒度**（仅 @ 我 / 全部 / 静音）
6. **文件全页阅读模式** + per-file 最后修改溯源
7. 线程视图、已读回执（产品决策项，IM 重功能）
